# LumaNox iOS 自测用例

本文档用于 **手动自测** LumaNox iOS 端已实现功能。与 [ios-parity-spec.md](./ios-parity-spec.md)（页面/UX 对齐）互补：本文只覆盖**当前可测的真实业务**，不含仍为 UI 壳的模块。

**最后更新**：与工程 `ios/README.md` 中 Scope 一致（含 PIN、保险箱、回收站、手动/自动备份、私密相机等）。

---

## 1. 测试环境

| 项 | 要求 |
|---|---|
| 系统 | iOS 16.0+（真机优先：相机、Face ID、文件书签） |
| 工具 | Xcode 15+；`cd ios && xcodegen generate` |
| 构建 | `xcodebuild -scheme LumaNox -project LumaNox.xcodeproj -sdk iphonesimulator -destination 'platform=iOS Simulator,name=iPhone 16' build` |
| 账号/网络 | 无需联网（订阅未接）；备份到 iCloud 需登录 iCloud |
| 对比机（可选） | Android 同版本，用于 `backup.dat` / `.aivb` 交叉验证 |

### 1.1 测试前准备

- [ ] 删除 App 后重装，或确认可接受清空数据
- [ ] 系统设置中允许：相机、麦克风（录像）、相册、Face ID（若测生物识别）
- [ ] 「文件」App 中预先创建文件夹，例如：`我的 iPhone/AIVault` 或 iCloud Drive 下 `AIVault`
- [ ] 准备 2～3 张测试照片、1 段短视频（系统相册）

### 1.2 结果标记

| 符号 | 含义 |
|---|---|
| P0 | 阻塞发布 |
| P1 | 重要功能 |
| P2 | 一般 / 体验 |
| ✅ / ❌ / ⏭ | 通过 / 失败 / 跳过 |

## 2. 启动与 PIN 锁（P0）

| ID | 用例 | 步骤 | 预期结果 | 优先级 |
|---|---|---|---|---|
| L-01 | 首次启动强制设置 PIN | 全新安装 → 打开 App | Splash 后进入 6 位 PIN **设置**（1/2、2/2），未完成不能进主页 | P0 |
| L-02 | PIN 不一致 | 设置阶段两次输入不同 | 提示不匹配，可重新输入 | P0 |
| L-03 | PIN 解锁 | 设置成功后杀进程重开 | 进入解锁页，正确 PIN 进入四 Tab | P0 |
| L-04 | PIN 错误 | 连续输入错误 PIN | 错误计数提示；仍无法进入主页 | P0 |
| L-05 | 修改 PIN | 设置 → 安全 → 修改 PIN | 旧 PIN 校验通过后设新 PIN；新 PIN 可解锁 | P1 |
| L-06 | Face ID / Touch ID | 设置 PIN 后按提示开启生物识别 | 解锁页可用生物识别进入主页 | P1 |
| L-07 | 后台锁屏 | 解锁后进主页，切后台 **>60s** 再回前台 | 再次要求解锁（PIN/生物识别） | P1 |
| L-08 | 备份密钥缓存 | 解锁后进入 **设置 → 备份与同步** | 可进行「立即备份」（不要求再次输入备份专用密码） | P1 |
| L-09 | 首启 RestoreLogin | 已授权自动备份目录且存在 `backup.dat`，**删除 App 重装**（或清除 Keychain 后重装） | Splash 后进入「欢迎回来」恢复页，输入**原备份 PIN** 一次即可恢复并进入主页 | P0 |
| L-10 | 重装选文件夹恢复 | 重装后无书签，在 PIN 设置页点「从已有备份恢复」→ 选含 `backup.dat` 的文件夹 | 进入 RestoreLogin；正确 PIN 恢复成功 | P1 |
| L-11 | 放弃备份 | RestoreLogin 连续输错 PIN **≥3 次** | 出现「放弃备份，创建新相册」；确认后回到全新 PIN 设置（1/2） | P1 |

## 7. 手动备份与恢复（P0）

| ID | 用例 | 步骤 | 预期结果 | 优先级 |
|---|---|---|---|---|
| B-01 | 未解锁备份 | 杀进程后未解锁 → 手动导出 | 提示需先解锁 / 备份密钥未缓存 | P1 |
| B-02 | 手动导出 | 解锁 → 设置 → 备份与恢复 → **导出备份** → 选位置存 `.aivb` | 进度完成；结果页显示项数与大小 | P0 |
| B-03 | 空保险箱导出 | 无媒体时导出 | 失败提示保险箱为空 | P2 |
| B-04 | 手动恢复 | 导出后删 App 重装 → 设**相同 PIN** → 从备份恢复选 `.aivb` | 输入 PIN → 恢复成功；媒体回到保险箱 | P0 |
| B-05 | 恢复 PIN 错误 | 恢复时输入错误 PIN | 提示密码不正确，保险箱不写入 | P0 |
| B-06 | 包格式 | 用文本编辑器看文件头（可选） | 以 `AIVAULT` magic 开头（二进制） | P2 |

### 7.1 与 Android 交叉（P1，双机）

| ID | 用例 | 步骤 | 预期结果 |
|---|---|---|---|
| B-X1 | Android → iOS | Android 手动备份 `.aivb`，同 PIN 在 iOS 恢复 | 媒体恢复成功或跳过已存在项 |
| B-X2 | iOS → Android | iOS 导出 `.aivb`，在 Android 恢复 | 同上 |

## 8. 自动备份到「文件」App（P0）

| ID | 用例 | 步骤 | 预期结果 | 优先级 |
|---|---|---|---|---|
| A-01 | 选择文件夹 | 设置 → 备份与同步 → **选择备份文件夹** | 选本地或 iCloud 下目录；显示路径；提示已关联 | P0 |
| A-02 | 生成 backup.dat | 关联后立即备份或点「立即备份」 | 「文件」中该目录出现 **`backup.dat`**（覆盖更新） | P0 |
| A-03 | 开关关闭 | 关闭「启用自动备份」 | 冷启/后台不应再写（可隔日观察无新时间戳） | P1 |
| A-04 | 开关打开 | 再次打开自动备份 | 可再次立即备份；设置页显示上次备份时间 | P1 |
| A-05 | 取消授权 | 「取消文件夹授权」 | 状态变为未关联；立即备份提示未授权 | P1 |
| A-06 | 备份页状态 | 备份与恢复页顶部卡片 | 已关联显示「已就绪」与路径；未关联显示未关联 | P2 |
| A-07 | PIN 变更后同步 | 修改 PIN 后等待后台任务（或杀进程重开解锁） | `backup.dat` 更新（时间戳变化）；旧 PIN 无法恢复新包 | P1 |
| A-08 | 恢复后同步 | 手动从 `.aivb` 恢复成功后 | 若已关联文件夹，应触发一次自动同步（与 Android 一致） | P2 |
| A-09 | 冷启补跑 | 上次自动备份 **>8h** 后冷启 App（已解锁、已关联） | 约 30s 后可能更新 `backup.dat`（不保证精确，观察文件修改时间） | P2 |
| A-10 | 未解锁跳过 | 重启后不进解锁，放后台 | 不应崩溃；未解锁时不写外部文件 | P1 |

### 8.1 文件检查（可选）

在「文件」App 打开备份目录：

- [ ] 存在 `backup.dat`
- [ ] 不应长期残留 `backup.dat.writing`（异常中断后重开 App 应被清理）
- [ ] 恢复测试：首启用 L-09/L-10；手动包仍可用设置内「从备份恢复」测 `.aivb`
