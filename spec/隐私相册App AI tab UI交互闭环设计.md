# 隐私相册App 标准版MVP UI交互闭环设计

核心前提：MVP交互遵循「极简、直观、无学习成本」，所有AI功能自动触发为主、手动操作为辅，贴合普通用户使用习惯，同时强化「隐私安全」感知；双端交互保持一致（iOS适配底部tab、Android适配顶部/底部tab均可），闭环覆盖「进入\-使用\-操作\-反馈\-留存」全流程。

核心功能交互闭环：AI敏感内容识别→AI自动分类→AI垃圾照片清理→AI隐私脱敏，四大功能联动，无冗余操作，每个功能均有「触发\-执行\-反馈\-后续操作」完整链路。

## 一、整体UI框架（双端通用）

底部3个核心tab（极简布局，MVP不做多余入口）：

1. **首页（推荐）**：默认进入，展示「全部照片（时间线）」\+ 顶部AI快捷入口（脱敏、清理），简洁无冗余

2. **分类相册**：AI自动生成的分类文件夹（截图/证件/人像/风景/美食/文档），点击进入对应分类

3. **私密相册**：需指纹/面容/数字密码解锁，存放AI识别的敏感内容，独立加密空间

顶部通用控件：搜索框（轻量化文字搜图）、设置按钮（仅保留「隐私设置」「AI设置」2个核心入口）。

## 二、四大核心AI功能 交互闭环（详细可落地）

### 功能1：AI敏感内容自动识别（核心差异化，自动触发）

#### 交互闭环：触发→提醒→操作→反馈→归档

1. **触发条件**（自动，无需用户操作）：
        

    - 用户首次安装App，授权相册权限后，App后台静默扫描全部本地照片（弹窗提示：「AI正在扫描敏感内容，保护您的隐私安全」，带加载进度条）；

    - 用户新增照片（拍照/保存/导入），实时触发AI识别（后台静默，无明显感知，不卡顿）；

    - 识别范围：证件（身份证、银行卡）、账单截图、聊天隐私截图、票据、隐私人像。

2. **提醒交互**（轻量不打扰）：
        

    - 识别到敏感内容后，底部弹出轻量通知（非系统通知，App内弹窗），持续3秒，可点击：「发现3张敏感照片，是否移入私密相册？」，带「立即移入」「稍后处理」两个按钮；

    - 若用户点击「稍后处理」，敏感内容暂存于「首页\-待处理敏感照片」入口（顶部小图标，带数字角标，提示未处理数量）；

    - 首次使用时，弹窗引导：「开启AI敏感识别，自动保护您的隐私照片」，仅提示1次，可勾选「不再提示」。

3. **操作交互**：
        

    - 点击「立即移入」：弹窗提示「正在加密移入私密相册」（1秒反馈），完成后弹出「已安全移入，可在私密相册查看」，点击「去查看」跳转至私密相册对应文件夹；

    - 点击「待处理敏感照片」：进入列表页，展示所有未处理敏感照片，顶部可「全选/取消全选」，底部有「移入私密相册」「忽略（标记为非敏感）」按钮；

    - 忽略后，AI后续不再识别该类照片为敏感（可在「设置\-AI设置」中重置识别规则）。

4. **反馈与归档**：
        

    - 移入成功后，首页对应照片自动标记「已加密」（小锁图标，仅用户可见），系统相册中不显示该照片（仅App内可见）；

    - 私密相册内，敏感内容按「证件/账单/隐私人像」自动分组（复用AI分类能力），便于查找；

    - 无敏感内容时，首页不显示「待处理」入口，后台持续静默监测。

### 功能2：AI自动相册分类（基础体验，自动生成）

#### 交互闭环：自动生成→展示→操作→编辑

1. **自动生成**（无感知，后台同步）：
        

    - 与AI敏感识别同步触发，扫描相册后，自动生成6个核心分类文件夹（截图/证件/人像/风景/美食/文档），无需用户手动创建；

    - 新增照片时，实时自动归类到对应文件夹（例：拍照的人像自动进入「人像」分类，截图自动进入「截图」分类）；

    - 分类规则可在「设置\-AI设置」中调整（例：隐藏「美食」分类、合并「文档/票据」分类）。

2. **展示交互**：
        

    - 点击底部「分类相册」tab，进入分类列表页，每个分类以「封面图\+分类名称\+照片数量」展示（封面图为该分类下AI筛选的最清晰照片）；

    - 分类排序：常用分类（截图、人像）排在前，不常用分类（文档、证件）排在后，可手动拖拽调整顺序；

    - 分类文件夹右上角有「更多」按钮（三个点），点击可操作「重命名、隐藏、删除分类」。

3. **操作交互**：
        

    - 点击任意分类，进入该分类照片列表（时间线排序，最新在前）；

    - 长按单张照片，可进行「移动分类、移出分类、删除、加密到私密相册」操作；

    - 批量操作：点击右上角「编辑」，可全选/多选照片，进行批量移动、删除、加密。

4. **编辑与反馈**：
        

    - 移动分类时，弹窗展示所有分类选项，选择后即时反馈「已移动至XX分类」；

    - 隐藏分类后，该分类不在列表页展示，可在「设置\-AI设置」中重新显示；

    - 删除分类时，弹窗提示「删除分类不会删除照片，照片将回归「全部照片」」，避免误操作。

### 功能3：AI垃圾照片清理（高粘性，主动触发\+自动提醒）

#### 交互闭环：提醒→扫描→筛选→清理→反馈

1. **触发与提醒**：
        

    - 自动提醒：AI扫描相册后，若发现模糊、废片、过曝、重复连拍、无用截图（如临时验证码），首页顶部弹出「发现可清理垃圾照片，共X张」，带「去清理」按钮；

    - 主动触发：首页顶部快捷入口点击「AI清理」，直接进入清理页面；

    - 定期提醒：每7天自动弹窗提醒1次（可在设置中关闭），提升日活。

2. **扫描与筛选交互**：
        

    - 进入清理页面，AI自动扫描并分类展示可清理照片，按「重复照片、模糊废片、无用截图」分组；

    - 重复照片：AI自动标记最优1张（清晰、构图好），其余标记为「可清理」，展示「对比图」（点击可查看原图差异）；

    - 模糊/无用截图：AI自动勾选可清理照片，用户可手动取消勾选（避免误删有用照片）；

    - 顶部可切换分组，查看不同类型的可清理照片，勾选框支持「全选/取消全选」。

3. **清理操作交互**：
        

    - 勾选完成后，点击底部「一键清理」，弹窗二次确认：「确认清理选中的X张照片？清理后可在最近删除中恢复（保留30天）」；

    - 点击「确认」，开始清理，展示清理进度条（快速完成，不超过2秒）；

    - 支持「取消清理」（清理过程中点击取消，已清理的照片不恢复，未清理的停止）。

4. **反馈与后续**：
        

    - 清理完成后，弹窗反馈「清理成功！释放X MB存储空间」，带「查看最近删除」「返回首页」按钮；

    - 清理后的照片进入「首页\-最近删除」文件夹，保留30天，可随时恢复；

    - 无垃圾照片时，清理页面显示「相册很干净！AI将持续监测」，提升用户体验。

### 功能4：AI隐私脱敏（轻量化，手动触发\+自动适配）

#### 交互闭环：触发→脱敏→预览→分享/保存→反馈

1. **触发场景**（手动为主，自动为辅）：
        

    - 场景1：分享照片时，自动触发脱敏提示（仅敏感照片）；

    - 场景2：单张照片详情页，点击右上角「脱敏」按钮，主动触发；

    - 场景3：批量选中照片，底部点击「脱敏」，批量处理。

2. **脱敏交互**：


    - 触发后，AI自动识别照片内敏感信息（手机号、身份证号、二维码、人脸），1秒内完成脱敏；

    - 脱敏效果：手机号/身份证号用「\*」打码（例：138\*\*\*\*1234、110101\*\*\*\*1234），人脸用模糊马赛克，二维码直接涂抹；

    - 支持「自定义脱敏」：脱敏后，点击照片上的脱敏区域，可选择「取消脱敏、更换打码样式（马赛克/模糊/涂抹）」；

    - 脱敏不改变原图：脱敏后的照片生成副本，原图仍加密保存，避免误操作。

3. **预览与操作**：
        

    - 脱敏完成后，自动进入预览页，展示「原图vs脱敏图」对比（左右滑动切换）；

    - 预览页底部有「保存脱敏图、分享脱敏图、重新脱敏」三个按钮；

    - 分享脱敏图时，直接调用系统分享（微信、QQ、短信等），无需额外操作，分享后弹窗反馈「已分享脱敏版，保护隐私不泄露」。

4. **反馈与留存**：
        

    - 保存脱敏图后，自动存入「分类相册\-脱敏照片」文件夹，便于后续查找；

    - 首次使用脱敏功能时，弹窗引导：「AI隐私脱敏，分享照片不泄露敏感信息」，仅提示1次；

    - 脱敏失败（如无敏感信息），弹窗提示「未检测到敏感信息，无需脱敏」。

## 三、全App交互闭环（串联四大功能，提升留存）

1. **用户首次使用闭环**：安装→授权相册权限→AI扫描（敏感识别\+分类）→弹窗提醒敏感内容→引导移入私密相册→引导查看分类相册→提示AI清理功能→完成引导，进入首页（全程不超过1分钟）；

2. **日常使用闭环**：新增照片→AI自动识别（敏感则提醒）→自动分类→定期提醒AI清理→分享照片时自动脱敏→私密相册加密管理→后续可在分类/私密相册查找、编辑；

3. **操作反馈闭环**：所有操作（移入、清理、脱敏、分类）均有即时反馈（弹窗/Toast），无无效操作；误操作可通过「最近删除、取消脱敏、重置分类」挽回；

4. **隐私安全感知闭环**：私密相册解锁提示、脱敏分享反馈、敏感识别提醒、加密标记，全程强化「隐私安全」认知，提升用户信任度。

## 四、双端适配注意事项（开发落地重点）

1. iOS端：底部tab适配iOS原生样式，私密相册解锁优先用Face ID/Touch ID，弹窗样式贴合iOS原生设计（无圆角阴影，简洁）；

2. Android端：底部tab可自定义样式（与App整体风格统一），解锁支持指纹/面容/数字密码，适配不同机型的相册权限（Android 13\+用PhotoPicker）；

3. 性能适配：所有AI操作（识别、脱敏、清理）均在后台静默执行，不卡顿、不发热，千元机也能流畅运行；

4. 隐私合规：所有操作均在本地完成，弹窗明确告知用户「AI本地处理，不上传任何照片」，提升信任。

## 五、MVP交互核心原则（不偏离）

1. 不做复杂操作：所有AI功能「自动触发为主，手动操作为辅」，用户无需学习，上手即会；

2. 不堆冗余入口：仅保留核心tab和功能入口，避免用户混淆；

3. 强化隐私感知：每一步隐私相关操作（加密、脱敏）都有明确反馈，让用户感受到安全；

4. 预留扩展空间：交互架构兼容后续进阶功能（文搜图、精选集、伪装相册），避免后期重构。
