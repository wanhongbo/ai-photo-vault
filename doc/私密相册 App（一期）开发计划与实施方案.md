# 私密相册 App（一期）开发计划与实施方案

文档目标：基于《私密相册 App（一期）核心技术方案及实现说明》，输出可执行的开发计划与落地方案，用于研发排期、任务拆解、测试验收与提审准备。

## 一、一期目标与范围

### 1.1 产品目标

- 上线一款面向海外市场的私密相册 App（iOS + Android），核心价值为：本地离线、安全加密、便捷分享。
- 一期聚焦 MVP，优先保障可上线、可审核、可稳定运行。

### 1.2 一期范围（In Scope）

- 解锁与安全模块（PIN/图案锁、生物识别、后台自动锁定）
- 系统相册读取、照片导入与本地 AES 加密存储
- 私密拍照（直拍直存，不入系统相册）
- 相册管理（创建/重命名/删除、批量移动删除、回收站恢复）
- AI 打码（离线推理 + 模糊/马赛克）
- 海外社交分享（支持主流平台）
- 多语言国际化（首批 8 种语言）
- 备份与恢复（加密 ZIP 导出/导入）
- 内购付费（订阅 + 买断）

### 1.3 非一期范围（Out of Scope）

- 云同步、跨设备自动同步
- Web 管理端
- 高级 AI 编辑能力（如自动美化、目标替换）

## 二、总体实施方案

### 2.1 技术架构与分层

- 表现层：Flutter 页面 + GetX Controller
- 领域层：核心用例（导入加密、打码、备份恢复、付费校验）
- 数据层：Repository + Isar/Hive + Secure Storage + File Storage + Plugin Adapter
- 核心层：加密能力、权限网关、任务调度（Isolate）、错误码与日志

### 2.2 关键技术决策

- 状态管理：GetX（轻量，便于快速迭代）
- 存储方案：优先 Isar（关系与查询能力更强），也可按团队熟悉度选 Hive
- 密钥方案：`flutter_secure_storage` 管理主密钥，禁止本地明文落盘
- 加密算法：AES-256-CBC（照片），SHA-256（口令存储）
- 性能方案：耗时任务统一放入 Isolate，列表分页 + 缩略图缓存
- 合规方案：最小权限申请，隐私政策与权限文案提前本地化

### 2.3 数据模型建议

- `Album`：id、name、coverId、createdAt、updatedAt
- `PhotoAsset`：id、albumId、encryptedPath、thumbPath、metadata、createdAt、deletedAt
- `TrashItem`：photoId、expireAt（30 天）
- `SecuritySetting`：lockType、pinHash、biometricEnabled、failCount
- `SubscriptionState`：plan、expireAt、entitlements
- `BackupRecord`：filePath、createdAt、version、checksum

## 三、里程碑计划（8 周）

### W1：项目基建与安全底座

- 初始化工程与目录规范，配置环境区分（dev/prod）
- 接入核心插件，完成基础路由与异常处理框架
- 完成密钥生成与读取、密码哈希能力
- 输出：可运行骨架版本（双端）

### W2：认证门禁与权限链路

- PIN/图案锁 + 生物识别接入
- App 生命周期监听，后台自动锁定
- 相册只读权限申请与引导文案
- 输出：进入 App 的完整鉴权流程

### W3：导入加密与私密拍照

- 系统相册分页读取、缩略图缓存、批量选择
- 导入后加密写入私有目录并落库
- 私密拍照直拍直存（不写系统相册）
- 输出：导入/拍摄/加密/展示主链路可用

### W4：相册管理与回收站

- 自定义相册 CRUD
- 照片预览、批量移动与批量删除
- 回收站 30 天恢复机制
- 输出：管理闭环可用

### W5：AI 打码能力

- TFLite INT8 模型接入与离线推理
- 人脸/证件/手机号区域打码（模糊/马赛克）
- 新图生成策略（不覆盖原图）
- 输出：识别-处理-保存流程可用

### W6：分享、备份恢复、多语言

- 系统分享面板适配，临时文件自动清理
- 加密 ZIP 导出/导入与密码校验
- i18n 全页面覆盖（英/中简繁/日/韩/法/德/西）
- 输出：海外发布核心能力齐备

### W7：内购与权益控制

- 月订阅/年订阅/买断接入
- 免费版与高级版能力开关
- 本地状态恢复 + 平台凭证校验
- 输出：商业化链路可灰度验证
- **说明（原生双端）**：Android / iOS 内购统一经 **RevenueCat** 接入，与商店商品、Entitlement 在后台对齐；详见《私密相册 App（一期）原生双端架构设计方案》及 `doc/android/10-内购付费.md`、`doc/ios/10-内购付费.md`。

### W8：稳定性冲刺与提审

- 性能优化（首屏、滚动、内存、包体积）
- 审核材料准备（隐私政策、权限说明、截图/录屏）
- 回归测试与 blocker 修复
- 输出：提审版本（iOS/Android）

## 四、任务拆解建议（按模块）

### 4.1 安全与解锁

- 锁屏页与解锁状态机
- 生物识别失败降级逻辑
- 连续错误计数与闯入抓拍触发

### 4.2 导入与加密

- 相册资产分页读取
- 图片复制、加密、索引写入事务
- 批量导入进度与失败重试

### 4.3 媒体与相册管理

- 相册 CRUD 与封面更新
- 预览页手势交互
- 回收站定时清理任务

### 4.4 AI 打码

- 模型加载与推理 pipeline
- 区域后处理与渲染策略
- 打码强度参数化

### 4.5 商业化与增长

- 订阅页与权益说明
- 购买恢复与状态同步
- 免费额度限制（张数/次数/水印）

## 五、测试与验收方案

### 5.1 测试策略

- 单元测试：加密解密、口令哈希、权限状态机、权益判断
- 集成测试：导入 -> 加密 -> 相册展示 -> 打码 -> 分享全链路
- 真机回归：iOS 12+、Android 8.0+ 主流机型

### 5.2 验收标准（一期）

- 批量导入 100 张照片成功率 >= 99%
- 加密、AI 打码过程中 UI 无明显卡顿（主线程不阻塞）
- 前后台切换锁定无遗漏
- 回收站恢复与过期清理逻辑准确
- 内购购买、恢复购买、权益生效稳定

## 六、风险与预案

- 审核风险：权限申请文案与实际用途严格一致，避免被拒
- 兼容风险：插件锁定稳定版本，建立最小可回滚方案
- 性能风险：大图懒加载、缩略图缓存、耗时任务 Isolate 化
- 数据风险：密钥仅存安全硬件，备份包强制密码与校验
- 体验风险：关键操作增加状态反馈（进度、成功、失败重试）

## 七、资源与人力建议

- Flutter 开发：2 人（功能实现 + 双端兼容）
- 测试：1 人（功能回归 + 审核场景专项）
- 产品：0.5 人（验收、文案、定价）
- 设计：0.5 人（多语言 UI 与提审素材）

## 八、交付物清单

- 可提审安装包（iOS/Android）
- 隐私政策与权限说明文档（英文优先）
- 测试报告与缺陷闭环清单
- 发布说明（版本功能、已知问题、后续计划）
