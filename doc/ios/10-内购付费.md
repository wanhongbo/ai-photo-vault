## 技术方案要点

### 统一方案：RevenueCat

双端内购均通过 **[RevenueCat](https://www.revenuecat.com/)** 接入：由 RevenueCat SDK 对接底层 **StoreKit**（含 StoreKit 2 路径，以 SDK 版本为准），在 Dashboard 与 Android **共用同一项目** 的 **Entitlements / Offerings / Products**。

### iOS SDK 与配置

- 依赖 **RevenueCat iOS SDK**（Swift Package Manager 或 CocoaPods）；在应用启动时 **`Purchases.configure(withAPIKey:)`**，使用 **App Store 应用** 对应的 **Public API Key**（可与 Android 不同 key，同属一 RevenueCat 项目）。
- 在 App Store Connect 配置订阅组与商品，与 RevenueCat 中 Product ID 绑定；**StoreKit Configuration** 文件可用于本地调试（与 RevenueCat 测试流程配合）。
- 购买：获取 **`Offerings`** → **`purchase(package:)`** 等 → 从 **`CustomerInfo`** 读取 **Entitlement** 是否激活。
- **恢复购买**：**`restorePurchases`**；必要时结合 **App Store**「重新下载已购项目」体验说明。

### 权益与 Domain

- 与 Android 相同：Domain 仅依赖 **Repository** 输出的「是否享有某 Entitlement」；命名与 Dashboard 一致。
- **隐私清单（Privacy Manifest）**：按 RevenueCat 官方说明声明其 SDK 收集项；与隐私政策一致。

### 与直接接 StoreKit 的关系

- 业务层不直接驱动 `Product.purchase` 主路径；订阅状态以 RevenueCat 为准。调试时仍可在 Xcode 中验证交易与 RevenueCat 用户绑定。

### 隐私与合规

- 不向 RevenueCat 上传用户相册或密钥；在隐私政策中披露 RevenueCat 用于订阅管理。

---

## 分层落位（参考）

- **Platform**：RevenueCat 初始化与购买/恢复封装。
- **Domain**：与 Android 对齐的 entitlement 与免费版限制逻辑。
- **Data**：可选本地权益缓存。

### 一致性

- 与 Android **同一 RevenueCat 项目**、同一 Entitlement 体系；避免双端条款与开关不一致。
