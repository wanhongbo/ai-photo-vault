## 技术方案要点

### 权限

- 目标 API 33+ 使用细分媒体权限（如 `READ_MEDIA_IMAGES`）；更低版本与分区存储按官方兼容策略处理。
- 若仅「每次选图」而不浏览全库，优先 **Photo Picker**（`ActivityResultContracts.PickMultipleVisualMedia` 等），减少对「持续读取外部存储」的依赖。
- 若产品需要按相册/时间轴浏览系统库，再申请相应读权限并在首次请求前展示用途说明（与海外审核要求对齐）。

### 数据访问

- `MediaStore` + `ContentResolver.query`，按日期字段排序；相册桶可用 `BUCKET_ID` / `BUCKET_DISPLAY_NAME` 分组展示。

### 展示与性能

- Paging 3 + Compose Lazy 列表；缩略图使用 **Coil** 等组件加载 `content://` URI，或 `ContentResolver.loadThumbnail`；大图按需解码，避免一次加载全分辨率。

### 安全

- 只读系统相册，**不调用** `MediaStore` 写入或删除用户原图。
