# Luma Vault — 支付相关功能开发计划与配置清单

**范围**：Android 端应用内购买（**自动续订订阅** + **一次性买断**）、RevenueCat 集成、权益判定、Paywall / 订阅管理 UI、与 **Google Play** 及 **隐私/Data Safety** 对齐。  
**技术选型**：与 [10-内购付费.md](./10-内购付费.md) 一致 — **RevenueCat** 统一对接 **Google Play Billing**（业务层不直接使用 `BillingClient` 主流程）。  
**一期权益边界**：以产品文档为准；**不包含**伪装图标、闯入抓拍等与付费绑定的能力（参见 [Luma-Vault-一期-Android-开发计划.md](../Luma-Vault-一期-Android-开发计划.md) 第一节）。

**关联代码现状（编写时基线）**

- `BuildConfig.REVENUECAT_API_KEY`：由 `android/local.properties` 中 **`revenuecat.apiKey.android`** 注入（**勿**将真密钥提交 git）；`app` 已依赖 `purchases`。  
- `SubscriptionState` / `SubscriptionStateEntity`：Room 单例表已存在，可继续接入 **同步 RevenueCat 结果** 的写入路径。  
- `PaywallScreen`：已接 RevenueCat Offerings；设置页「订阅」仍为占位入口时可链至 Paywall。

### 文案与费用：客户端不写死，依赖 RC / 商店返回


| 类型                      | 要求                                                                   | 数据来源（实现时）                                                                                                                                                          |
| ----------------------- | -------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **价格、货币、计费周期文案**        | **禁止**在代码或 `strings.xml` 中写死金额（如 `$4.99`）、「约 ¥xx/月」等                 | Google Play 经 RevenueCat 下发的 `StoreProduct` / `Package` / `SubscriptionOption` 上的 **本地化价格与周期**（SDK 字段名以当前 `purchases` 版本为准，如 formatted price、subscriptionPeriod 等） |
| **单条商品（订阅 / 买断）的标题与说明** | **禁止**把具体套餐营销长文案仅写在客户端资源里当作唯一来源                                      | 优先使用商店返回的 **商品标题、描述**（`StoreProduct` 本地化字段）；若需完全远程化营销层，使用 **RevenueCat Paywalls** 或 **Offering / Package metadata**（与 Dashboard 配置一致）                              |
| **Paywall 上的权益列表**      | 若产品与运营要求「改字不发版」，应走 RC Paywalls / 远程配置，**不得**依赖仅本地 `strings.xml` 的长列表 | RC 远程模板或 metadata；客户端只保留极少量无营销风险的骨架文案（如「恢复购买」「加载失败」）                                                                                                               |
| **按钮上的动态拼句**            | 例如「以 {price}/月 订阅」中的 `{price}` 必须来自上表商店字段                            | 字符串模板用 `stringResource` + **运行时**传入的格式化价字符串                                                                                                                        |
| **错误与状态提示**             | 可本地化键值（取消、网络错误等），**不含**价格                                            | `strings.xml`                                                                                                                                                      |


**验收**：切换 Play 国家/货币或修改 Console 价格后，客户端 **无需发版** 即可显示新价格（在 RC 刷新商品成功后）；营销文案若走 Paywalls/metadata，同样以远程为准。

## 一、目标与交付物


| 目标  | 说明                                                                                          |
| --- | ------------------------------------------------------------------------------------------- |
| 可购买 | 用户可在 Paywall 选择 **月订 / 年订 / 买断** 并完成 Google Play 结算流程                                       |
| 可恢复 | 「恢复购买」可用，且与当前 Google 账号权益一致                                                                 |
| 可解释 | 错误码映射为用户可读文案（取消、网络、已拥有等）；**价格与套餐营销文案不写死**（见上文「文案与费用」）                                       |
| 可门控 | Domain 层统一判断 `isPremium` / 具体权益，列表导入、分享次数、水印、AI 配额等 **禁止在 Composable 内直接解析** `CustomerInfo` |
| 可上架 | Data Safety、隐私政策披露 RevenueCat；测试轨道验证通过                                                      |


**交付物**：可合并代码（SDK + Repository + UI + 门控用例）+ 本文 **第三节配置清单** 在控制台全部勾选完成 + 内购测试报告（含 **订阅** 续费/取消、**买断** 购买与恢复、恢复购买全场景说明）。

## 三、配置清单（执行时逐项勾选）

下列 **建议名称** 可直接复制到 Play / RevenueCat；若已上架或已有付费用户，**勿改** 商品 ID 与 Entitlement identifier。客户端字符串需与 **§3.0 名称总表** 一致。

### 3.0 名称总表（Play ↔ RevenueCat ↔ 代码）


| 用途                           | 建议填写值                                 | 出现位置                                                                   |
| ---------------------------- | ------------------------------------- | ---------------------------------------------------------------------- |
| 月订 — Play 订阅商品 ID            | `luma_vault_premium_monthly`          | Play「订阅」；RC Product identifier                                         |
| 年订 — Play 订阅商品 ID            | `luma_vault_premium_annual`           | Play「订阅」；RC Product identifier                                         |
| 买断 — Play **一次性商品** ID（非消耗型） | `luma_vault_premium_lifetime`         | Play「应用内商品 / 一次性商品」；RC Product identifier                              |
| 月订 — Base plan ID            | `luma-vault-monthly-base`             | 仅 Play（该订阅下的 base plan）                                                |
| 年订 — Base plan ID            | `luma-vault-annual-base`              | 仅 Play                                                                 |
| Entitlement identifier       | `premium`                             | RC Entitlement；**月 / 年 / 买断** 三个 Product 均 attach 到此，任一有效即 `isPremium` |
| Offering identifier          | `default`                             | RC Offering；代码 `offerings.current` 或 `offerings["default"]`            |
| RC Package lookup key（月）     | `$rc_monthly`                         | RC Package，绑定月订 Product                                                |
| RC Package lookup key（年）     | `$rc_annual`                          | RC Package，绑定年订 Product                                                |
| RC Package lookup key（买断）    | `$rc_lifetime`                        | RC Package，绑定一次性 Product（RevenueCat 标准 Lifetime 包名）                    |
| RC Project 名（展示）             | `Luma Vault`                          | Dashboard                                                              |
| RC Android 应用名（展示）           | `Luma Vault Android`                  | Dashboard                                                              |
| Android 包名 `applicationId`   | `**com.xpx.vault`**（与当前仓库一致）或自改后全链路统一 | Play、RC、Gradle                                                         |


**商品结构**：**两条订阅**（月 / 年，各一条 + 一个 Base plan）+ **一条一次性买断**（非消耗型，与订阅共用同一 Entitlement `premium`）。客户端从 `Offering.current.availablePackages` 渲染三档，**价格与标题均来自商店经 RC 下发**。

**产品注意**：若用户先订订阅再购买断，需与运营约定是否提示「可在 Play 取消订阅」以免双重付费；实现层仍以 `entitlements["premium"]?.isActive` 为准。

## 四、测试验收清单（支付专项）


| 场景              | 预期                                         |
| --------------- | ------------------------------------------ |
| 冷启动无网           | 使用 Room 上次快照展示；恢复网络后可 `sync` / 打开 App 触发刷新 |
| 新购月订            | `CustomerInfo` entitlement active；门控全部放开   |
| 新购年订            | 同上                                         |
| 新购买断（一次性）       | `premium` active；设置页展示与订阅区分（无「续费」误导）       |
| 恢复购买            | 换机或清除数据后恢复成功（**含**已购买断的恢复）                 |
| 订阅用户再买断         | 按产品策略提示是否取消订阅；权益仍为 active                  |
| 用户取消支付对话框       | 无崩溃；提示「已取消」或静默                             |
| 已是会员点购买         | 友好提示或走管理订阅                                 |
| 订阅过期（测试时钟或沙盒加速） | 门控回落免费策略                                   |
| `prod` 构建       | 无 RC Debug 日志洪泛；无密钥打印                      |
